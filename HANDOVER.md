# 鍛冶屋シミュレーター 引継ぎ資料

本資料は、今後の開発やアップデートに向けた技術的な引継ぎ事項をまとめたものです。

## 1. UIとスタイリングの構造
現在のUIは **Tailwind CSS** と **グラスモーフィズム**（すりガラス効果）を基調としています。
- **カスタムCSS**: `<style>` タグ内に `.bg-glass`, `.bg-glass-light`, `.bg-glass-panel`, `.bg-image` などの独自クラスを定義しています。
- **背景画像の切り替え**: `App` コンポーネント内の `getBgImage` 関数で、`gameState` (現在の画面状態) に応じて背景画像を動的に切り替えています。
- **UIコンポーネント**: パネル類には `bg-glass` または `bg-black/60` といった半透明の背景色と `backdrop-blur` を組み合わせて適用しています。新しく画面や要素を追加する際も、このスタイルを踏襲すると統一感が出ます。

## 2. マスターデータの追加・修正
ゲームバランスの調整や新要素の追加は、`index.html` 内の上部にあるマスターデータ（定数オブジェクト）を直接編集することで行えます。
- **`MATERIALS`**: 新しい素材（ドロップアイテム）の追加
- **`RECIPES`**: 新しい武器・防具のクラフトレシピの追加
- **`ENCHANTS`**: エンチャントの種類と効果量・付与確率の重み付け
- **`BOSSES`**: ボスの出現日数や強さの調整
- **`RANDOM_EVENTS`**: 日替わりランダムイベントの追加

## 3. セーブデータの仕様
ゲームの進行状況は、ブラウザの `localStorage`（キー: `blacksmith_save_v22`）に自動保存されています。
もしセーブデータの構造（新しいステータス項目の追加など）を大きく変更する場合は、読み込みエラーを防ぐためにキー名を `blacksmith_save_v23` などに変更するか、マイグレーション処理を入れることを推奨します。

## 4. GitHub Pagesの更新
本プロジェクトはGitHubのリポジトリ (`https://github.com/ohchinchin/kajiya1.git`) と連携しています。
今後コードを修正したり画像を差し替えた場合は、以下のコマンドでPushするだけで自動的にGitHub Pages側も更新されます。
```bash
git add .
git commit -m "アップデート内容"
git push origin main
```
※反映まで数分かかる場合があります。
